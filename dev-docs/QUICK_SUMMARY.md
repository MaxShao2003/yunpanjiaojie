## 🎯 访问历史系统（核心特性）

### 位置
- 只显示在 `index.html` 的右侧栏
- 题目："最近访问"
- 副标题："交接阅读记录"

### 功能
✅ **自动记录**：所有内容页自动注册
✅ **去重机制**：同页面不重复，新访问更新时间
✅ **最多12条**：保留最新的12条记录
✅ **持久存储**：localStorage 保存（刷新不丢失）
✅ **类型识别**：自动推断页面类型（图标显示）
✅ **时间格式化**："刚刚" / "5分钟前" / "下午3:14" 等
✅ **可点击跳转**：点击记录快速返回该页面
✅ **清空按钮**：可选的"清空记录"功能

### 示例显示
```
🎬 采访视频          刚刚
📊 考勤表            5分钟前
📔 日记1             2小时前
📄 电话记录          今天 15:20
🔐 真实名单          昨天 23:45
```

## 💼 工作流程（对开发者）

### 如果要修改某个页面的访问记录显示

**方式1** - 自动（推荐）：什么都不用做，系统会从 `<title>` 自动提取

**方式2** - 自定义：在 `<body>` 标签上添加属性
```html
<body data-page-id="mypage"
      data-page-title="自定义标题"
      data-page-type="folder">
```

**支持的 type 值**：
- `folder` - 📁 文件夹
- `txt` - 📄 文本
- `xlsx` - 📊 表格
- `pdf` - 📋 PDF文档
- `jpg` - 🖼️ 图片
- `mp4` - 🎬 视频
- `admin` - 🔐 后台系统
- `system` - ⚙️ 系统页面
- `diary` - 📔 日记
- `audio` - 🎤 音频

## ✨ 为什么这个设计很好

1. **零侵入性**：几乎不改动原有代码，保护世界观
2. **自动化**：无需为每个页面配置，即插即用
3. **沉浸感强**：访问记录看起来像企业系统功能，不是外挂组件
4. **可追踪**：帮助玩家理清探索思路，但不强制
5. **持久性**：浏览器关闭后数据仍保留
6. **可视化**：图标 + 时间 + 标题，信息密度高
