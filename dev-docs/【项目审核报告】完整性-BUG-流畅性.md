## 三、已发现的Bug清单

### 🔴 重 - 关键功能缺陷

**Bug #1：五连击解锁条件过宽泛**
- 当前：仅需 `hasSeenWechatLog` 或 `assetsUnlocked`
- 应该：应该同时检查是否读过P23日记和P26覆写代码
- 影响：玩家可能在未获得充分线索时意外进入后台
- 修复：修改 `app.js` 第74-75行

```javascript
// 当前代码
const hasKey = localStorage.getItem('hasSeenWechatLog') === 'true' ||
              localStorage.getItem('assetsUnlocked') === 'true';

// 建议改为
const hasKey = localStorage.getItem('hasReadDiary3') === 'true' &&
              localStorage.getItem('adminEntryUnlocked') === 'true';
```

### 🟡 中 - 可用性问题

**Bug #2：P35 超时强制跳转未实现**
- 需求：停留120秒后自动跳转P36
- 当前：只显示倒计时，未触发重定向
- 影响：玩家可永久卡在P35页面
- 修复：在P35添加

```javascript
setupPageTimeout(120000, 'bad_end.html'); // 120秒后跳转
```

**Bug #3：P35 DESTROY 输入框提示不清晰**
- 问题：玩家可能看不到"如需维护操作，请输入指令"的输入框
- 建议：提高提示的可见性，或添加"输入 DESTROY 可触发真结局"的暗示

**Bug #4：进度百分比计算不精确**
- 当前：以 `/40` 计算，但项目实际有42个html文件
- 建议：明确定义哪40个主要页面参与进度计算

### 🟢 轻 - 体验优化

**Bug #5：P15 密码错误重试无提示梯度**
- 文档建议：连错3次后给更强的提示"试试他在聊天里反复抱怨的那句话"
- 当前：每次都是相同的错误提示
- 建议：增加重试次数追踪

**Bug #6：P19 密码提示信息不够递进**
- 文档建议：有分阶段的提示按钮（点一次看工号、点二次看猫名）
- 当前：只有一条提示
- 建议：改进为多级提示系统

**Bug #7：P30 茶水间监控画面过于简洁**
- 当前：只显示一团黑色像素和"信号受损"
- 建议：加入更多视觉干扰（闪烁、杂音、扭曲）增强恐怖感

### 🔵 信息 - 功能完整性

**Note #1：localStorage 状态变量不完整**
- 已使用的标签：`isLoggedIn`, `currentUser`, `visitedPages`, `progress`, 等
- 缺失的标签：`hasReadDiary3`, `adminEntryUnlocked` 在代码中未被主动设置
- 建议：在每个关键页面加入 `Puzzle.recordVisit()` 调用

**Note #2：登录逻辑过于宽松**
- 当前：任何非空账号密码都能登录
- 建议：保留这个宽松设定（符合游戏概念），但可添加特殊账号彩蛋
  - 如输入 "admin/OVERRIDE_998" 直接跳转后台
