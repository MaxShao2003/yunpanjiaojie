# 《交接云盘》四条谜题链完整跳转表

这份文档把主流程拆成四条核心谜题链，并补上推荐状态变量、页面依赖关系与防卡关设计。你可以直接把它交给 Cursor 或自己按这个逻辑写跳转。

## 一、建议使用的状态变量

为避免页面之间靠“猜”，建议用 `localStorage` 记录以下状态：

| 变量名 | 类型 | 作用 |
| --- | --- | --- |
| `hasLoggedIn` | boolean | 是否完成入口登录 |
| `hasSeenWechatLog` | boolean | 是否看过微信截图 |
| `assetsUnlocked` | boolean | 是否解开 `/assets` |
| `hasSeenClawPic` | boolean | 是否看过抓痕照片 |
| `hasSeenVentPic` | boolean | 是否看过排风扇照片 |
| `hasSeenDirectory` | boolean | 是否看过通讯录 |
| `hasSeenWeibo` | boolean | 是否看过微博 |
| `privateUnlocked` | boolean | 是否解开 `/private_zy` |
| `hasReadDiary3` | boolean | 是否读到第3篇日记 |
| `hasReadRules` | boolean | 是否读到真实规则 |
| `hasAdminCode` | boolean | 是否拿到 `OVERRIDE_998` |
| `adminEntryUnlocked` | boolean | 是否允许五连击进入后台 |
| `hasSeenYourDeskCam` | boolean | 是否看过你的工位监控 |
| `destroyAvailable` | boolean | 是否允许在最终页输入 `DESTROY` |

## 二、全局主流程骨架

```text
P01 登录
 -> P02 主界面
    -> P04 关于鼎铭
    -> P05 公共文件夹 -> P06/P07/P08/P09/P10
    -> P11 项目文件夹 -> P12/P13/P14/P15/P16 -> P17 -> P18

    -> P20 微博外链

P35 分流 -> P36 / P37 / P38
P36/P37/P38 -> P39 -> P40
```

## 三、谜题链A：解锁 `/assets`

目标：让玩家从“普通加班”滑进“前同事可能出事了”。

| 步骤 | 当前页 | 玩家行为 | 获得线索 / 答案 | 页面反馈 | 下一页 |
| --- | --- | --- | --- | --- | --- |
| A1 | `P02` | 点击 `项目文件夹 /projects` | 进入项目目录 | 打开项目列表 | `P11` |
| A2 | `P11` | 先查看普通项目文件 | 看到 `P12` 红衣反光、`P13` 异常报销、`P14` 诡异脚本 | 玩家开始怀疑 | 返回 `P11` |
| A3 | `P11` | 点击 `沟通记录_微信截图.jpg` | 得到口头禅 `不想干了` | 设置 `hasSeenWechatLog=true` | `P16` |
| A4 | `P16` | 读聊天截图 | 推导拼音答案 `buxiangganle` | 页面可提示“返回素材库尝试” | 返回 `P11` 或 `P15` |
| A5 | `P11/P15` | 点击 `素材归档 /assets` 并输入密码 | 正确答案 `buxiangganle` | 设置 `assetsUnlocked=true` | `P17` |
| A6 | `P17` | 查看抓痕照片 | 前同事遭遇异常的实锤 | 设置 `hasSeenClawPic=true` | `P18` |
| A7 | `P18` | 查看排风扇照片 | 红布与 `P12` 人影呼应 | 设置 `hasSeenVentPic=true` | 返回 `P02` |

### 这条链的情绪目标

- `P12-P14` 是“奇怪但可解释”
- `P16` 是“赵宇真的很想逃”
- `P17-P18` 是“赵宇已经不是单纯辞职了”

### 防卡关建议

- 如果玩家在 `P15` 连输错三次，页面底部可追加一句：`试试他在聊天里反复抱怨的那句话。`

## 四、谜题链B：解锁 `/private_zy`

目标：让玩家跨页面整合信息，正式进入赵宇留下的求生线。

| 步骤 | 当前页 | 玩家行为 | 获得线索 / 答案 | 页面反馈 | 下一页 |
| --- | --- | --- | --- | --- | --- |
| B1 | `P02` | 注意到 `赵宇私密文件夹 /private_zy` 被加锁 | 提示需要 `工号 + 猫名` | 无 | `P19` |
| B2 | `P19` | 查看提示 | 需要两段信息 | 无 | 返回 `P02` |
| B3 | `P02/P05` | 去公共文件夹查看通讯录 | 得到赵宇工号 `DM-8975` | 设置 `hasSeenDirectory=true` | `P09` |
| B4 | `P02` | 点击 `赵宇的个人微博（已离职）` | 准备找猫名 | 打开外链页 | `P20` |
| B5 | `P20` | 查看微博内容 | 得到猫名 `黑豆 / heidou` | 设置 `hasSeenWeibo=true` | 返回 `P19` |
| B6 | `P19` | 输入 `DM-8975heidou` | 解锁成功 | 设置 `privateUnlocked=true` | `P21` |
| B7 | `P21-P26` | 连续阅读 | 获得后台入口、规则、覆写代码 | 设置 `hasAdminCode=true` | `P26` |

### 这条链的情绪目标

- 先让玩家有一点“侦探感”
- 再让玩家发现，自己越了解赵宇，越像走进赵宇的命运

### 防卡关建议

- `P19` 输入框下可做一个逐步提示按钮：
  - 第一次点：`工号可以在公开资料里找到`
  - 第二次点：`猫的名字不在云盘里`

## 五、谜题链C：进入隐藏后台

目标：完成世界观翻面，把“交接人”变成“猎物”。

| 步骤 | 当前页 | 玩家行为 | 获得线索 / 答案 | 页面反馈 | 下一页 |
| --- | --- | --- | --- | --- | --- |
| C1 | `P23` | 读到第3篇日记 | 知道 `Logo 五连击` 可以进后台 | 设置 `hasReadDiary3=true` | `P24` |
| C2 | `P24-P26` | 继续阅读 | 在 `P26` 明确拿到 `OVERRIDE_998` | 设置 `hasAdminCode=true`、`adminEntryUnlocked=true` | `P26` |
| C3 | 任意前台页 | 连续点击左上角 Logo 5 次 | 触发隐藏入口 | 黑屏 1-2 秒，加静电音 | `P27` |
| C4 | `P27` | 输入 `admin / OVERRIDE_998` | 登录后台 | 成功进入后红黑 UI 替换前台风格 | `P28` |
| C5 | `P28` | 依次查看后台条目 | 得到“员工是材料”的完整真相 | 可访问 `P29-P32` | `P29/P30/P31/P32` |

### 这条链的情绪目标

- `P23-P26` 是“赵宇在托付最后的活路”
- `P27-P28` 是“原来这一切真的是系统化生产”
- `P31` 是整条链的峰值，必须让玩家感到“轮到我了”

### 防卡关建议

- 没读到 `P26` 前，Logo 五连击只给一个轻微震动，不直接开门
- 读到 `P26` 后，五连击才真正生效

## 六、谜题链D：结局分流

目标：在高压状态下让玩家做出最终选择，并给出三种味道完全不同的结局。

| 步骤 | 当前页 | 玩家行为 | 条件 | 页面反馈 | 下一页 |
| --- | --- | --- | --- | --- | --- |
| D1 | `P31` | 看到自己的工位被监控、倒计时开始 | 无 | 设置 `hasSeenYourDeskCam=true` | `P32` |
| D2 | `P32` | 查看自己的档案 | 无 | 强化“你已被标记” | `P33` |
| D3 | `P33` | 警报触发 | 无 | 自动跳转 | `P34` |
| D4 | `P34` | 读 HR 消息 | 无 | 人格化威胁完成 | `P35` |
| D5 | `P35` | 点击 `前往HR办公室` | 无 | 直接进入坏结局 | `P36` |
| D6 | `P35` | 点击 `拔掉网线，砸碎窗户逃跑` | 无 | 进入普通结局 | `P37` |
| D7 | `P35` | 在隐藏输入区输入 `DESTROY` | 玩家已到后台并读过关键文件即可默认允许 | 进入真结局 | `P38` |
| D8 | `P35` | 停留超过 120 秒无操作 | 无 | 播放敲门声，强制坏结局 | `P36` |

### 三种结局的作用

- `P36 坏结局`：告诉玩家制度会把顺从者完整吞掉
- `P37 普通结局`：你逃掉了，但系统可能永远不会真正结束
- `P38 真结局`：你对系统造成了伤害，但仍然没能保证它彻底死透

## 七、推荐的进度百分比变化

可在 `P02` 右上角显示 `当前交接进度`，随着关键节点提升，增强“验收程序”感。

| 节点 | 进度建议 |
| --- | --- |
| 初次进入主界面 | 12% |
| 看完公共资料 2-3 页 | 28% |
| 解开 `/assets` | 43% |
| 看完抓痕与排风扇照片 | 51% |
| 解开 `/private_zy` | 63% |
| 读完 `P24 true_rules` | 78% |
| 进入后台 `P28` | 87% |
| 到达 `P35 final_choice` | 100% |

这能和规则第6条形成直接呼应：`当交接进度达到100%时，绝对不要去HR办公室。`

## 八、推荐的最小实现顺序

如果你想先做出一个能完整跑通的版本，优先级按下面来：

1. `P01 -> P02 -> P05 -> P09 -> P11 -> P15 -> P16 -> P17 -> P18`
2. `P19 -> P20 -> P21 -> P22 -> P23 -> P24 -> P25 -> P26`
3. `Logo 五连击 -> P27 -> P28 -> P31 -> P33 -> P34 -> P35`
4. `P36/P37/P38 -> P39 -> P40`

这样就算你先不做 `P04 P06 P07 P08 P10 P12 P13 P14 P29 P30 P32`，也能先做出一条最小可玩链路。
