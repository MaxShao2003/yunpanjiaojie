# 《交接云盘》可直接复制给 Cursor 的分页面提示词包

## 使用方式

1. 先把下面的“通用母提示”发给 Cursor。
2. 然后按页面逐条发送对应的单页提示词。
3. 如果同一会话太长，先重新贴一遍“通用母提示”，再贴当前页面提示词。

## 通用母提示

```text
你现在是一个资深前端工程师，正在帮我制作一个静态网页 ARG 游戏，项目名叫《交接云盘》。

项目设定：
- 题材：企业微恐 / 新怪谈 / 伪造的公司 OA 云盘
- 玩家身份：新入职剪辑师，接手前员工赵宇的离职交接资料
- 隐藏真相：公司不是传媒公司，而是把员工当作“材料”的异常饲养场

技术要求：
1. 使用 HTML + CSS + 原生 JavaScript
2. 所有页面都是静态页面，通过 window.location.href 或 a 链接跳转
3. 前台系统使用蓝白灰企业风格，后台系统使用黑红暗灰风格
4. 页面代码结构清晰，便于后续复用
5. 所有文案使用中文
6. 尽量抽出可复用布局，例如顶部栏、侧边栏、文件列表、弹窗、假表格、假播放器
7. 可以创建共享文件，例如 styles/main.css、styles/admin.css、scripts/app.js、scripts/puzzle.js
8. 页面要尽量拟真，像真的企业云盘或 OA 系统

全局设计要求：
- 前10页要尽量正常，不要一开始就恐怖
- 恐怖信息藏在制度语言里，比如“交接”“验收”“留存”“状态稳定”
- 常出现的时间锚点：03:14、今晚24:00前、22:00后
- 左上角公司 Logo 需要支持“3秒内连续点击5次进入后台登录页”的隐藏交互，但只有玩家读到相关线索后再真正生效
- 可以用 localStorage 记录游戏状态，例如是否看过微信截图、是否解锁私密文件夹、是否拿到后台覆写代码

跳转说明：
- 每页都要明确写出当前页会跳到哪些页面
- 如果页面有密码输入，正确和错误反馈都要做好
- 如果页面是目录页，要用文件列表形式展示，而不是文章列表

输出要求：
- 直接给我完整代码
- 如果需要新增共享 CSS / JS 文件，也一起给出
- 页面中的中文文案请按我后续给你的单页说明来写，不要自己改世界观
```
