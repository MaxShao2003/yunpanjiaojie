## 🎮 快速开始

### 首次游玩建议

**不含剧透的最小操作：**
1. **登录** → login.html（任意账号密码，如 `test/test`）
2. **进入主界面** → 开始探索
3. **第一个谜题** → 点击左上角公司 Logo
4. **避免卡关** → 本地保存指南，遇到谜题可以参考

### 游戏的三个阶段

| 阶段 | 页面范围 | 目标 | 氛围 |
|------|---------|------|------|
| **第一阶段：正常职场** | P01-P10 | 探索，习惯系统 | 正常、内容驱动 |
| **第二阶段：线索浮现** | P11-P26 | 解开谜题，发现异常 | 恐怖、悬念递增 |
| **第三阶段：翻面与抉择** | P27-P40 | 进入真相，做出选择 | 绝望、紧张 |

   - 工号来自 P09 通讯录：`DM-8975`
   - 猫名来自 P20 微博：`黑豆` 拼音 `heidou`
   - 连接：全小写，无空格，无特殊符号
   - 正确格式：`DM-8975heidou`

3. **后台登录**（P27）：
   - 账号：`admin`
   - 密码：`OVERRIDE_998`（来自P23日记）

### 问题 2：五连击不生效

**症状**：Logo 五连击后没有进入后台

**解决**：
1. 确认已读过 P23（diary_03.html）
2. 确认在 3 秒内快速点击 5 次
3. 尝试在不同页面（P02 或 P05）重试
4. 清除 localStorage 重新开始（如果状态混乱）

### 问题 3：进度条不更新

**症状**：完成某个关键节点但进度没变

**解决**：
1. **手动刷新**页面，进度应该自动计算
2. **检查 localStorage**：打开浏览器开发者工具 → Application → localStorage
3. **验证这些变量**：
   - `visitedPages` 应包含访问过的页面
   - `assetsUnlocked` 应为 true （解开/assets后）
   - `adminLoggedIn` 应为 true （登录后台后）
4. 如果变量缺失，重新访问对应页面

### 问题 4：P35 最终抉择倒计时卡住

**症状**：倒计时不动了 或 不会自动跳转

**解决**：
1. 倒计时是 JavaScript 实现，确认浏览器没禁用 JS
2. 如果真的卡住，手动点击按钮强制跳转
3. 如果要测试 120 秒超时功能，保留该页面不动即可

### 问题 5：看不到 DESTROY 输入框

**症状**：P35 页面下方没有维护输入区

**解决**：
1. 向下滚动，输入框在底部，灰显状态
2. 输入框通常是隐晦的，用 CSS `opacity: 0.3`
3. 将鼠标悬停在下方，透明度会增加到 0.6

### 问题 6：输入 DESTROY 后没跳转

**症状**：输入了 DESTROY 但没有进入 P38

**解决**：
1. **检查前置条件**：必须已经登录过后台（P27 → P28）
2. **检查变量**：`localStorage.getItem('adminLoggedIn')` 应为 `'true'`
3. **检查拼写**：DESTROY 必须全大写
4. **检查键盘**：确认按了回车键
5. 如果都没问题，可能需要刷新浏览器重试

### 问题 7：某个页面无法加载

**症状**：点击链接后卡住或 404

**解决**：
1. **检查文件名拼写**：页面链接是否正确
2. **检查文件存在**：确认文件在 pages/ 目录中
3. **清除浏览器缓存**：Ctrl+Shift+Del 清除缓存后重试
4. **检查路径**：相对路径应为 `../pages/xxx.html`
5. **服务器问题**：确认在正确的服务器路径下运行

## 📊 三种结局对比

### 结局1：坏结局（P36）✗

**触发条件**：
- 在 P35 选择 `前往HR办公室`，或

有人替你整理好了工位。

你拔掉网线，抄起显示器砸碎了窗户。

- 你逃脱了加工厂，但系统无法被追踪
- 一切证据都被消灭

你在维护终端输入了 DESTROY。

你活着离开，但回头时仍看见更高处还有一层灯没熄。

系统回传：
第4号舱离线
其余舱体状态未知

**含义**：
- 你摧毁了一部分系统（第 4 号舱离线）

**玩家感受**：胜利但苦涩，希望但绝望，反击但无力
